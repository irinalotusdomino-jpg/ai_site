# AI Reels Creator — Hero Block

Односторінковий hero-блок: парюча AI-сфера, сітка з 4 вікон контенту
(2 фото + 2 відео), екран запуску AI, живе нейронне поле під курсором/пальцем.

## ⚠️ Точна структура файлів (щоб усе завантажилось)

Це найважливіша частина. Якщо фото/відео/лого не відображаються на сайті —
у 99% випадків причина саме тут.

```
<назва-репозиторію>/
├── index.html
└── assets/
    ├── logo.png        — лого
    ├── photo.png        — фото №1
    ├── photo2.png        — фото №2
    ├── reel.mp4          — відео №1
    ├── reel2.mp4          — відео №2
    ├── poster.png        — кадр-заглушка для reel.mp4 (поки відео вантажиться)
    └── poster2.png        — кадр-заглушка для reel2.mp4
```

**Правила, які треба дотримати буквально:**
1. Назви файлів — рівно такі, як тут написано, малими літерами.
   GitHub Pages чутливий до регістру: `Photo.png` ≠ `photo.png`.
2. Папка `assets` лежить **поруч** з `index.html`, в одному й тому ж
   репозиторії, на одному рівні (не всередині іншої папки).
3. Коли завантажуєш файли на GitHub — заливай папку `assets` цілком
   (перетягни саму папку в інтерфейс GitHub, а не файли з неї окремо;
   інакше GitHub може покласти їх у корінь без підпапки, і шляхи зламаються).
4. Не перейменовуй файли після завантаження без зміни `CONFIG` в
   `index.html` — шлях і назва файлу повинні співпадати 1:1.

Посилання в `index.html` прописані як **відносні** (`assets/photo.png`),
тому вони працюють однаково і локально, і на GitHub Pages — за умови,
що структура папок точно така, як вище.

## Як опублікувати на GitHub Pages

1. Створи новий репозиторій на GitHub (наприклад `ai-reels-creator`).
2. Завантаж у нього `index.html` і папку `assets/` **цілком**, зберігаючи структуру вище.
3. Відкрий **Settings → Pages**.
4. У полі **Source** вибери гілку `main`, папку `/ (root)` → **Save**.
5. Через 1–2 хвилини сайт буде на:
   `https://<твій-логін>.github.io/<назва-репозиторію>/`
6. Перевір у браузері: відкрий цю адресу і подивись, чи вантажаться фото/відео.
   Якщо ні — відкрий DevTools (F12) → вкладка **Network**, онови сторінку
   і подивись, які саме файли повертають 404. Це миттєво покаже, де саме
   розходження в назві/шляху.

## Що всередині hero-блоку

- **Екран запуску AI** — темний екран, блимає курсор, друкується текст,
  сканування, потім інтерфейс "збирається зі світла".
- **Живе нейронне поле** — частки й спотлайт реагують на курсор (десктоп)
  і на палець (мобільний/планшет), без блокування скролу сторінки.
- **AI-сфера** — парює замість мокапу телефону: дихає (2–3% пульсація),
  повільно обертається, при наведенні/дотику показує світні лінії,
  при кліку/тапі — хвиля світла, легко зміщується при скролі.
- **4 вікна контенту** — 2 фото + 2 відео у стилі міні-застосунків
  (кружечки-індикатори зверху + назва файлу), відео автоматично
  програються без звуку по колу.
- **Нижній рядок контактів** — email, TikTok, YouTube, другий YouTube-канал,
  Pinterest.

## Як редагувати контент

Усі тексти, кольори, шрифти, посилання, шляхи до файлів і налаштування
всіх ефектів знаходяться в об'єкті `CONFIG` на початку `<script>` в
`index.html`. Кожен параметр підписаний коментарем `/* ЗМІНИТИ */`.
Розмітку і CSS чіпати не потрібно — досить редагувати `CONFIG`.

Головні секції в `CONFIG`:
- `colors` — усі кольори сайту
- `fontHeading` / `fontBody` — шрифти з Google Fonts
- `logo`, `windows` (4 фото/відео), `socials` — шляхи до файлів і посилання
- `orb` — розмір, кольори (`colorA`/`colorB`), розмиття сфери
- `effects` — вмикання/вимикання кожного з трьох WOW-ефектів
- `boot` — тексти й швидкість екрана запуску AI
- `neural` — щільність часток нейронного поля
